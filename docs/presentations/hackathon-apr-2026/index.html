SKILL.md

The contract. Defines inputs, outputs, domain decisions, safety rules. The AI reads this; it never overrides it.

Python script

The implementation. Runs the pipeline, writes structured output. Must support --demo for instant testing.

Demo data

Synthetic test data bundled with every skill. Run a demo instantly without real patient data.

Report

Markdown + JSON. Human-readable and machine-parseable. Includes disclaimers and provenance.

Challenge tracks

Three tracks, one framework

Track 1: New skill development

Add a bioinformatics skill to ClawBio.

Variant annotation, pathway analysis, clinical reporting, or your own idea. The skill must conform to the SKILL.md template and include demo data.

Getting started:

cp templates/SKILL-TEMPLATE.md skills/your-skill/SKILL.md
mkdir -p skills/your-skill/tests skills/your-skill/examples
# Fill in SKILL.md, write tests, then implement

Judging: Does it solve a real problem? Is the SKILL.md complete? Does --demo work? Are there tests?

Track 2: Agent workflows

Chain multiple skills to solve a multi-step genomics question.

Example: full patient report

VCF annotator → PharmGx Reporter → PRS Calculator → Profile Report. One genome, four skills, one unified report.

Example: GWAS to drug target

GWAS Lookup → Fine-mapping → PubMed Summariser → ClinPGx. From association signal to actionable drug information.

Use the bio-orchestrator or build your own chaining logic. The goal: a workflow that produces something neither skill could produce alone.

Track 3: Equity and access

Audit representation gaps in genomic datasets.

Use the HEIM equity scoring framework to measure how well a dataset, tool, or database represents diverse populations.

Ideas

  • Score a public GWAS for ancestry representation
  • Compare PRS accuracy across populations
  • Audit ClinVar variant classification by ancestry
  • Build a dashboard showing equity gaps

Why it matters

  • 86% of GWAS participants are European
  • PRS lose up to 80% accuracy in non-European populations
  • AI amplifies existing bias
  • Measurement is the first step to fixing it

Bonus challenge

TuringDB Graph Challenge

Additional prizes for teams using TuringDB as a graph layer

What is TuringDB?

A high-performance, in-memory graph database built in C++. Column-oriented storage, zero-lock concurrency, and git-like version control for graphs.

200xfaster than Neo4j on deep queries
0.1msquery latency
10M+node graphs
8-hopqueries in 476ms
pip install turingdb

OpenCypher query language. Python SDK. REST API. github.com/turing-db/turingdb

Three graph directions

Drug repurposing

Use agentic AI + knowledge graphs to identify repurposed drugs. Focus on rare diseases and underrepresented patient populations.

Graph-based memory

Build a graph-powered memory layer for biomedical agents: R&D, clinical research, tumour boards, and more.

Multimodal KG integration

Fuse multiomics with clinical data and knowledge graphs inside agentic workflows. Applications in cancer research.

You can combine the TuringDB bonus with any of the three main tracks. Build a new skill that uses a graph backend, chain skills through a knowledge graph, or score equity across a graph of variant-population relationships.

How to submit

  1. Fork ClawBio/ClawBio on GitHub
  2. Create a branch named hackathon/your-team-name
  3. Add your skill under skills/your-skill-name/
  4. Open a PR before 16:30 with a description of what it does
  5. Demo live in 3 minutes: show the problem, run the skill, show the output

PR template: Title, track (1/2/3 + optional TuringDB), team members, one-paragraph summary, --demo command to reproduce.
